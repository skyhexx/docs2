Resolving Names

Resolving a domain name is process of retrieving a domain's records what saved in the domains. And there is no more other permission for user to access the records from the domain, All the records are public to everyone.

Resolver has a list of key-value data structure to store the records. The key and value are string in solidity, and the data didn't self-authentication format infomation, so you should validate the data format before saving and after retrieving.

Trust Domains supports many types of records:

  • Cryptocurrency addresses
  • IPFS CID
  • Metadata
  • Social Media

More type of records will coming soon!

Resolving cryptocurrency addresses#

To resolve a cryptocurrency address, you just supply both the domain namehash and cryptocurrency key of the cryptocurrency whose address you want to resolve. For example:

Cryptocurrency keys:

  • Bitcoin - wallet.BTC.address
  • Ethereum - wallet.ETH.address
  • Conflux - wallet.CFX.address
  • Zcash - wallet.ZEC.address
  • Neo - wallet.neo.address
  • Binance Smart Chain - wallet.bnb.address
  • HecoChain - wallet.bnb.address
  • OKExChain - wallet.okt.address
const tns = new TrustNameService()
const domain = 'token.cfx'
const tokenId = namehash(domain)
const btc = await tns.address(tokenId, 'wallet.BTC.address')
console.log(btc)
// 1GX28yLjVWux7ws4UQ9FB4MnLH4UKTPK2z
const eth = await tns.address(tokenId, 'wallet.ETH.address')
console.log(eth)
// 0x0efdcc214acfdda3603c4e0e6e567bb1b34569c1
const cfx = await tns.address(tokenId, 'wallet.CFX.address')
console.log(cfx)
// cfx:aaks65zn43jf4jwffcy0fyadspzh9uszpy9jdyhwdj
const zec = await tns.address(tokenId, 'wallet.ZEC.address')
console.log(zec)
// t1bEGpCsvYenFdw3YqSFJazJBefNGnY5TK3

Resolving IPFS#

To resolve a IPFS CID, you just supply both the domain namehash and IPFS key of the domain whose IPFS hash you want to resolve.

const tns = new TrustNameService()
const domain = 'token.cfx'
const tokenId = namehash(domain)
const ipfsHash = await tns.get(tokenId, 'ipfs.value')
console.log(ipfsHash)
// QmXnqs44mWCA9xid6wRH1v2G9JFvRisiBVPi5zKHsRZd2h

Resolving Social Information#

To resolve the social media information bind on the domains, you can pass a the social keys and the token id, you will get the social informations.

Social keys:

  • Twitter - social.twitter
  • Telegram - social.telegram
  • Blog - social.blog
  • Email - social.email
const tns = new TrustNameService()
const domain = 'token.cfx'
const tokenId = namehash(domain)
const twitter = await tns.get(tokenId, 'social.twitter')
console.log(twitter)
// https://twitter.com/trust_domains
const telegram = await tns.get(tokenId, 'social.telegram')
console.log(telegram)
// https://t.me/trustdomains
const blog = await tns.get(tokenId, 'social.blog')
console.log(blog)
// https://trustdomains.medium.com
const email = await tns.get(tokenId, 'social.email')
console.log(email)
// contract@trustdomains.org
