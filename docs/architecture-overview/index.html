Architecture Overview

Trust Domains are followed ERC721 token standard, so user can manage their domains by the compatible wallet which support ERC721 tokens, and the developers can integrate the domains to their DAPPs easily.

Trust Domains has four domains name system:

  • Conflux Name Service â€” This domains service deployed on Conflux blockchain. The domains with a subfix '.cfx'

  • BSC Name Service â€” This domains service deployed on Binance Smart Chain. The domains with a subfix '.bnb'

  • Heco Name Service â€” This domains service deployed on HecoChain. The domains with a subfix '.heco'

Name service architecture#

The domains service has two principal components: the Registry, and Resolver. Registry is a smart contract that contains a list of all domains and subdomains, and map the domains to the owner address and resolver address. Resolver is a map list contains the records associated with that domain.

The domains name service only has one Registry smart contract deployed on the blockchain. But different domains could associated with different Resolver. The registry is a ERC721 tokens, some every domains can be identified by a unique number, its tokenId, which is the namehash of a domain name.

The namehash algorithm is in here.

Registry#

Registry is the core smart contract for Trust Domains. The contract maintains the core data lists of a domains and subdomains, except the information in the ERC721, it's also stores:

  • Owner address
  • Resolver address
  • Domain name
  • Subdomain name

Registry extends the ERC721 methods, it has a set of methods to minting new domains, creating new subdomains. The owner of a domain may be either an external account or a smart contract. The owner can manage the domains easily with that methods. No one can control your domains without permission.

Owners of domains can do:

  • Set a resolver
  • Transfer domain ownership
  • Add a new subdomain
  • Burn a subdomain

Trust Domains only support second-level and third-level domains (e.g.: tom.wallet.cfx). This is to avoid the limitation of too long domain name.

Resolver#

Resolver is a core smart contract that is used for resolving and storing domain records. This is where a domain store their data, such as cryptocurrency addresses, and IPFS CIDs for decentralized websites.

Resolver define a list of key-value data structure. This structure allows users to store arbitrary records effectively. Also includes a set of methods for set and resolve the records, making it easier to access general information about the domains which the right resolver associated with it.

To learn more detail about resolver, you can check the source code of Resolver.sol
